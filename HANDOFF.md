## 锁定结论（勿重新论证 / 勿回退）
- 引擎 = **账本 + 软护栏**；裁判 = 方向与内容；脾性留 prompt；裁决 = **单 call · verdict-first**。
- ★**立场 = 独立存储 · 两轴**（轴 A 留你 ↔ 放你走 / 轴 B 随你 ↔ 离你），和「情」**无公式关系**；裁判按个刃角度叙事性挪存值、引擎只读值出 tell。（早期「引擎公式现算倾向」已废——**勿回退**。）
- 一台 **tell 引擎**统起 留逃 / 因果 / 溯行军；溯行军 = 立场被锁死的**反常离群点**、外来非本丸所出。
- 重连 = **三道 gate**：认回 → 契满 → 重结契（= 认回 ∧ 契满 ∧ 轴 A 放你走 ∧ 轴 B 随你走）。
- **明 / 暗双线平行**：明线 = 治国（地基）、**暗线 = 四程（觉醒 / 重连 / 留逃 / 溯行军）= 主线**；焊点 = §3.3 每裁决三盘；暗线不给进度条（靠 tell 感）。
- **推手非死轨**（§17.0 开放总则）：创意 defer/transfer + 治理出色 + 平衡钳制 三路可度；失衡监测的牙挂民心 / 稳定 / 派系，不挂「照办没」。
- **删读心**：无免费明牌机制；行为 tell 只在叙事自然流露（免费），主动探全归 **情报系统**（花钱 / roll / 派系化 · 主导失守 → 真但片面 · 非金手指）。
- **世界绝大多数是普通 NPC**（软态）；6 刀 + 溯行军是人海里零星几根针，刀坐镇 / 傅役 / 妃皆特例非默认。
- **储君 = 人**：政治倾向（账本载重 dial）+ 三观「等等角度」（软态裁判养）；人格由五源合熬，**将军本人的行动是大头**。
- **刀 = 三层 · 只中间一层要填**（§19）：canon 靠**刀帐世界书**（自带 · 不手写每把刀）/ **L2 此世维度池**（开局 roll · 问卷调味 · 作者填）/ 立场 · 情 · 契 · tell **游戏中长**（不预设 / 不锁存 / 不 roll）。**否定「预写 6 刀深档池」**（死板 / 冗余 / 追不上新刀）。
- **刀的此世执念 = 明线 agenda**（§9 末）：世界线分每刀一个此世角色目标（孩子做储君 / 派系上位 / 晋品 / 离开大奥…），刀为扮演主动追 = 不被抹杀 → active NPC；**焊死权谋 × 关系**（§3.3）、喂 §9 立场（不写死方向）、是限制；**不挂解密 · 不 gate §16 认回**（两线只在关系层交汇）。
- **★忠诚铁律**（§9）：你的刀对你 **100% 忠诚**（不背叛 / 通敌）—— 留 / 逃只是「怎样爱你才对」的**忠诚内分歧**（留派自私 vs 逃派无私）;派系战 = 忠诚之刀的内战。← 与 **溯行军**（非你之刀 · 伪装忠诚）根本区别。
- **★溯行军能力边界**（§22.4）：它 **无力改大世界**（改不了历史 / 规则）;核心能力 = **知道所有人的身份**（全场信息优势）→ 唯一杠杆 = **留住你**（= 它唯一能做的「改史」动作）。可疑圆满 = 信息优势精准操弄、**非改世界**。
- **tell 校准容错化**（R1）：校准归引擎（裁判只渲染措辞、不拿捏明显度）+ 分档范式库 + 一致性锚 + **读 tell = 可验证累积循环**（系统容忍单条模糊；§15.2 / §15.6）。
- **节奏 = 引擎软导演**（R5）：stall → 推 / beat 锚「攒够了」/ 步长引擎可感 / 程内限速 + 保供（火候留裁判 §14.4）—— **全确定性、无新 LLM 判断**（§17.4e）。
- **形态 = React standalone 产品先行**（§20 T3 · 当产品做 · Phase 1 纯前端 SPA + BYOK + 本地存档 + 接受可窥）；**酒馆卡 ST / 世界书 / MVU 降为将来可选适配**；**引擎核 + §14–19 全部机制平台无关、不因 React 改一字**（§20 T0 核 / 适配分层 · T1 承载 = A 混合 · T2 记忆 4 层 = 头号关切 · 史册收情感锁 · 分层叙事容三层分阶上）。§0–19 的 ST 词 = 适配层 ST 实现（主线 React 见 T1 对照）。
- **★schema 三层模型**（§21 岔口 ③ · 主原则）：**L1 简单维护变量**（引擎持久 · 最小原语）/ **L2 复杂派生变量**（引擎确定性派生 + 注入 · 不持久 · = 简单变量的数量关系 + 条件判定）/ **L3 软态**（AI 叙述）。**只 L1 进 schema · AI 不持久任何结构化状态**。
- **★连贯非掷骰**（答「AI 只表演怎么连贯」）：**引擎拥真相 / AI 拥讲法**;三柱 = 因果链（史册 ↔ 因果 · cite 缘起）+ 持有者锁死动机（= 阴谋的作者）+ AI 表演永锚（史册 J / tell 历史 / 滚动总结）。骰子只 jitter 幅度;方向翻转永远是上膛因果浮出（§14.1 门控）、非随机。因果确定性逻辑链 = arm/ripe/inject/fire/update（§21 岔口 ⑥ · AI 不写数、不越 ripe 集）。
- **★暗线 escalator + 因果库 + 多 call**：溯行军 **scheme-stage**（渗透 → 布局 → 收网 · schema E · ∥ 明线立嗣推手 · 补 §17.4e④ · 立场轴 A 仍锁死）;**§22.8 因果库 B5**（5 域 18 缘起种子 · seed 非穷举 · §22.5=持有者面 / §22.8=缘起面）;**多 call ≤ 3 共同维护推演**（§21 岔口 ②）。
