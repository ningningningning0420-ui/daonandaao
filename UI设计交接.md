## 2. UI 的本分（§20 T0 · 一句话定位）
引擎核 = 纯 TS · 平台无关 · 持有一切游戏逻辑（账本 / 软护栏 / 因果 / tell / 记忆）。**UI = 薄渲染层 + 输入采集**,不持游戏逻辑、不算后果、不存权威状态。UI 经**适配层接口**读引擎态 / 发玩家动作。→ **UI 怎么改都不动引擎核**（反之亦然）。

## 3. UI 铁律（违则破卡 · UI 设计的硬约束）
- ★**明 / 暗双线**（§17.4a）:**明线 = dashboard 可见**（治国三件 / 派系 / 养成 / 立嗣推手）;**暗线绝不上进度条**（觉醒 / 重连 / 留逃 / 溯行军 = 靠 tell + 基调感、玩家自己读）。给暗线做进度条 = 直接毁掉解密与基调。
- ★**删读心 / tell 不端答案**（§15）:UI **绝不显示**「倾向 = 逃 73」「X 是溯行军」「立场轴值」。**无任何隐藏态面板**。tell 只活在演绎散文里。
- ★**因果默认隐藏**（§14.4）:**无因果面板**。玩家靠 tell 推、靠情报窥探（§17.3）揭片段。
- ★**裁决 = 混合偏开放**（§3.3）:**锚**（可读可预览）+ **自由文本**（亲笔）二者并存。
- **NSFW = 呈现层开关**（§7 / §16.2）:连接演法可关 / 换皮肤接触、**机制不变**（契照推）。UI 一个开关切呈现。
- **步长 = 玩家旋钮**（§14.5 / §17.4e）:UI 给 日 / 月 / 季 控件;引擎给建议默认（危机硬锁日）,玩家可覆盖。
- **隐藏态完整性 Phase 1 接受可窥**（§20 T3）:纯前端 devtools 能看 state,**不防**（顺 §15.6 非强迫精神）。

## 4. 裁决幕 = UI×引擎唯一交界屏（verdict 契约已全锁 · UI session 第一块该做）
**一个裁决回合的数据流**:
1. **出幕**（引擎 + 裁判出）:纷争陈述 + **2–4 个立场锚**,每锚一行**方向感**（老中 ↗ / 外样 ↘ … 只给方向、不给幅度 / 不给翻不翻）。
2. **玩家**:**选一个锚 / 改写一个锚 / 自己写**（自由文本）。
3. **提交 → 裁决 call**（thinking 临场 + tool_use 出结构化 verdict）→ 引擎落账。**verdict 结构玩家看不到**。
4. **演绎 call**（流式散文）→ UI 渲染「这场戏」,**tell 织在散文里**（微表情 / 迟疑 / 可疑的圆满）。

**裁决幕 surface 什么 / 绝不 surface**:
- ✅ surface:纷争、锚 + 方向感、玩家输入（选 / 改写 / 自写）、演绎散文（流式）、**明线值变化**（dashboard 治国 / 派系数可动）。
- ❌ 绝不 surface:verdict 的 `deltas` 数值 / `因果` / `真蝴蝶` / `tell 指令` / 立场轴值 —— 全隐藏,玩家**从散文里读**。

**call 拓扑**（UI 要知道时序）:出幕 call → [玩家] → 裁决 call（thinking→tool_use）→ 引擎落账 + 派生 tell → 演绎 call（流式）。一回合 ≤ 3 call;轻量动作（批奏折等）走单 call 快结算。

## 8. 制作要求规格（UI session 硬约束 · 作者定 2026-06-16）
> 开 UI session 的**不可妥协制作标准**;配合 `ui-ux-pro-max` skill。**美学至上 —— 看起来简单基础 = 失败。**

**规格三连**:
- **游戏类型**:刀剑乱舞同人-策略后宫（经营 + 权谋 + 恋爱出路 + 解密 · LLM 驱动）。
- **美学风格**:刀剑乱舞风格-古典和风（大奥 / 江户 · 典雅 · 暗色描金 / 和纸质感 / 刀纹意象 · 见 §0 基调）。
- **包含内容**:一个**游戏主界面 + 相关派生子界面**（裁决幕 / 明线 dashboard / 动作菜单 / 关系层 / onboarding / 设置 · 见 §4–5）。

**形态 / 范围**:
- 复杂 web 端前端框架,**以 LLM 交互为基础的游戏** —— 把「对话 / 流式演绎 / 裁决亲笔输入」当**核心交互**纳入设计（非附属聊天框）。
- **只做前端原型,完全不做后端**（无 API / DB / 账号 · BYOK 接口留壳即可）。
- 主界面点击任一交互入口 → 至少展示**已设计好的模态框 / 标签页**（空壳也要成型;该屏若已定内容则带具体内容 —— 裁决幕 §4 已定数据流,优先做实）。

**硬性美学 / 工程要求**（逐条达标 · 违则视为失败）:
1. 除 logo / 视觉设计外 **全中文化**。
2. **禁用 emoji**,改用**美观、高级的 ICON**（专业 icon 库 · 线性 / 描金风为佳）。
3. **内部**警告提示 + 通知框（自绘组件 · **不用**浏览器原生 alert / notification / confirm）。
4. **现代排版**:非浏览器默认字体 —— 选高级中文字体 + 和风衬线,讲究字阶 / 行距 / 留白。
5. **微交互动画**:巧妙、灵动;悬停效果 + 交互反馈 + 状态过渡,界面「感觉灵敏生动」、鼓励交互。
6. **高端 / 最先进设计,避免简单 MVP**;**尽量避免占位符**（用拟真内容）。
7. **性能 / 可测**:语义化 HTML5 元素 · 所有交互元素带**唯一描述性 ID**（便于浏览器测试）· 优化快加载。
8. 充分发挥 **`ui-ux-pro-max`** 的能力 —— **表现普通的前端不可接受**。
