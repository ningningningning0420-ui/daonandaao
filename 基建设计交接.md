# 刀男大奥 · 基建系统设计交接（新 session 开场白 brief）

> 给「基建专题 session」的开场白。**canonical = 同目录 `设计文档.md`;本文件 = 这块要谈什么、接哪、别碰什么。**
> 用法:新 session 先跑 `/daonandaao` 对齐(HANDOFF + 设计文档 + 记忆),再读本文件,然后开谈。纯设计、无代码,走 brainstorm → 写 `设计文档.md` → commit。

## 一句话
为计划书补一块缺失机制:**基建（设施升级）+ 任命 + NPC 忠心**,并把**经济压力与政治压力焊成一个「联合压力场」**(非两条独立表)。

## 为什么开这块（缺口诊断 · 2026-06-16 作者点出）
- **现有**:§8 政治 / 派系 / 继承 / 养成 · §22.2 经济物理（三件互锁 / 四季 / 死亡螺旋）· §17.3 情报系统 · §14.4 失衡监测。
- **缺**:① **设施升级系统**——全无;② **任命机制**——§8.3 只一句「任命 = 把刀 / 人放进派系位置」、**未机制化**;③ **NPC 忠心值 → 行为**——无（§22.6.4 D4 有 NPC「真实站队」、但没有忠心**数**驱动行为 / 产出）。
- **未联合**:经济压力（§22.2）与政治压力（§8.1 推手 / §14.4 失衡）现在**各跑各的**,作者要它们**共同作用**。

## 作者要的 4 条（原话精炼 · = 设计目标、非已定结论）
1. **联合压力场**:经济压力 + 政治压力 = 共同作用的**一个**压力场。
2. **基建升级系统（参照审神者不妙屋）**:所有设施升级都有**正面效果**,但**融入政治逻辑**——升级不是纯增益,带政治重量 / 代价 / 涟漪。
3. **重要职位任命由将军做 → 影响设施产出**:任命对象的**派系 + 忠心**决定产出与偏置。例 ——**情报系统主事者若有派系且非 100% 忠诚 → 必有倾向**(情报被其派系立场染色)。
4. **NPC 忠心值 → 行为**:NPC 对将军的忠心影响其行为。

## 焊点 / 接哪（事实 · 新 session 别重造）
- **联合压力场的两侧已存在**:**§22.2 经济物理** = 经济侧;**§8.1 推手 + §14.4 失衡监测** = 政治侧。任务 = 把两者统成一个场(**扩**这两节、别另起炉灶)。
- **★中枢假设（待新 session 验证 / 精化 · 非锁定）**:**基建 + 任命 = 经济 × 政治的耦合点**。每个设施产经济(国库 / 兵戈 / 民心 / 情报),但由一个**被任命的人**主理,那人有派系 + 忠心(政治)→ ∴ **每个设施都是经济与政治交汇的节点**。作者举的「情报主事者派系 → 情报偏置」正是这耦合的样板 → 这很可能就是联合压力场的**物理实体**。
- **§17.3 情报系统已埋钩子**:「派系化主导 · 主导失守 → 真但片面」。**任命 = 决定谁握主导**的机制 = 这条已有设计的**上游**;情报主事者 = 一个 appointee 实例。这块设计要和 §17.3 对齐、把那个「主导」接到任命上。
- **§7 纳妃 tradeoff + §8.3 刀坐镇 + §8.5 刀当傅役**:任命**你的刀**去主理设施 = 类似 §7「纳 = 该刀真退出前朝职能」的机会成本(刀被「用」在职位上)→ **任命与关系层咬合**。刀 appointee = 100% 忠诚(§9)、无偏置;NPC appointee = 真忠心、有偏置 → 任命 = 「忠诚但占用一把刀 vs 高效但可能被染色」的取舍。
- **§5 精简状态网 + §1 普通 NPC 软态**:若每个 NPC 都存忠心数 = 撞「精简」。**候选解(待拍)**:只有**被任命 / 坐镇的 NPC** 升格为「被追踪」槽(带 `{派系, 忠心, → 产出修正, 偏置}`),泛 NPC 仍软态 —— **任命 = 把一个 NPC 提进追踪槽**。

## 要 brainstorm 的岔口（提问、不预答 · 一次一口）
1. **联合压力场怎么「联合」**:交叉耦合(空国库 → 恩赏给不动 → 派系更快不满;派系坐大 → 截留经济产出)? 共享危机阈? 一个派生复合压力量(∥ §17.4b)? 还是「基建 + 任命节点」本身就是耦合载体?
2. **基建升级「全正面但带政治」的政治逻辑**:升级惠及某派? 需某派支持 / 拨款才能升? 升完移动派系倒向? 造出依赖 / 把柄? —— 那个「不妙屋式纯正面」与「政治代价」如何并存。
3. **任命机制的形态**:任命 = 裁决级动作? appointee 的 `{派系, 忠心}` → 产出修正 + 偏置方向;**偏置怎么显现**(克扣 / 筛情报 / 挪款 / 拖延 / 阳奉阴违)?
4. **NPC 忠心 = 存值还是派生**:落 schema(哪些 NPC)? 怎么映到行为? 与 §22.6.4 D4 站队 的关系? 与刀 100% 忠诚(§9)如何区分、不混淆?
5. **设施清单**:哪些设施(情报 / 财政 / 军备 / 工造 / 民政 / 教化…)、各产什么(挂 A 治国三件 / 情报量 / 兵戈)、升级有几层、层级门槛。
6. **硬 / 软切口 + schema 落点**:设施 + 任命 + appointee 忠心 多半是**新 L1 硬态** → 会扩 §14.3 A–J + §21 岔口 ③ 的 zod schema。**好消息**:§21 ③ 刚定的 migrate-on-load 迁移框架正好接住 schema 增长、老存档不丢。

## 锁定红线（既有不变量 · 新 session 不许违反）
- **§9 忠诚铁律**:你的刀对你 **100% 忠诚**(留 / 逃是忠诚内分歧)——**NPC 忠心是另一根轴**,别和刀的立场混为一谈。刀 appointee 无背叛偏置;**只有 NPC appointee 才有忠心 < 100% 的偏置**。
- **§5 精简状态网**:只有**载重**的进账本——别为每个泛 NPC 开忠心栏(见上「候选解」=只追踪被任命者)。
- **§14.3 硬 / 软切口 + §21 三层 schema**:新态分 L1(持久原语)/ L2(派生注入)/ L3(软态)入位;只 L1 进 schema。
- **§14.4 + §22.2 扩、非复制**:联合压力场长在既有失衡监测 + 经济物理上。
- **协作 feedback 红线**:别塞机械配额(最多 / 至少 N 条);给世界观事实让 AI 收敛;常驻块不放活状态(演化态归 MVU / schema、别冻成静态副本)。
